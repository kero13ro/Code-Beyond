# Web 動畫工具比較：Lottie vs Rive vs Spline vs Spine

動畫在現代 Web 設計中不只是裝飾，它是使用者體驗的一部分。傳統 GIF 太重、CSS 動畫寫起來繁瑣、影片格式缺乏彈性，這就是專門動畫工具存在的原因。

以下比較四個主流動畫工具：Lottie、Rive、Spline 與 Spine。

## 快速比較

| 功能 | Lottie | Rive | Spline | Spine |
|------|--------|------|--------|-------|
| **類型** | 2D 向量 | 2D 互動 | 3D | 2D 骨骼 |
| **檔案格式** | JSON | .riv | .spline | JSON |
| **透明背景** | ✅ | ✅ | ❌ | ✅ |
| **互動性** | 基本 | 進階 | 中等 | 中等 |
| **跨平台** | ✅ 優秀 | ✅ 良好 | ✅ 以 Web 為主 | ✅ 以遊戲為主 |
| **免費專案** | 無限制 | 3 個 | 無限制 | 試用版 |
| **主要用途** | Web／行動 | Web／App | 3D Web | 遊戲 |

## Lottie

### 是什麼？
由 Airbnb 開發，Lottie 是基於 JSON 的動畫函式庫。可以把它理解成**向量版的 GIF**。

### 優點
- **輕量**：JSON 檔案很小，載入快（通常 < 100KB）
- **跨平台**：支援 iOS、Android、Web、React Native
- **可縮放**：向量圖形，任何解析度都不失真
- **資源豐富**：[LottieFiles](https://lottiefiles.com/) 提供大量免費動畫
- **設計師友善**：可直接從 After Effects 匯出

### 缺點
- **互動能力有限**：主要用於播放，進階互動需額外開發
- **依賴 After Effects**：製作動畫需要熟悉 Adobe After Effects
- **效果限制**：部分 After Effects 功能無法完美轉換

### 適合
Loading 動畫、圖示動畫、插圖動畫、簡單的跨平台動畫。

[LottieFiles - 下載免費輕量動畫](https://lottiefiles.com/)

## Rive（前身為 Flare）

### 是什麼？
Rive 是為回應使用者行為的互動動畫而生的工具。

### 優點
- **強大的互動性**：
  - 滑鼠／觸控互動
  - 拖曳事件
  - 捲動觸發動畫
  - 基於狀態的動畫序列
- **透明背景**：與網頁無縫整合
- **跨平台**：支援 Web、iOS、Android、Flutter
- **內建編輯器**：不需要 After Effects

### 缺點
- **Timeline 控制有限**：無法用 JavaScript 精確控制進度——只能播放、暫停、重播（不能從中間點開始）
- **免費方案限制**：
  - 只有 3 個專案
  - 無法匯出可編輯的 .rev 檔案
- **無法轉移**：免費專案無法備份或移轉

### 適合
使用者互動動畫（如角色跟隨游標）、遊戲 UI 動畫、捲動觸發效果、App 微互動。

[Rive - 建立可在任何地方執行的互動動態圖形](https://rive.app/)

## Spline

### 是什麼？
Spline 是瀏覽器端的 3D 設計工具，用來建立 3D Web 內容。

### 優點
- **設計師導向**：直覺的介面，不需要寫 3D 程式碼
- **即時協作**：多人同時編輯專案
- **瀏覽器端操作**：不需安裝軟體
- **豐富的 3D 功能**：材質、光源、動畫 timeline
- **彈性匯出**：可匯出為程式碼、影片或圖片

### 缺點
- **不支援透明背景**：最大的限制——無法與頁面背景融合
- **必須占滿容器**：3D 場景需要佔據整個容器
- **效能需求高**：3D 動畫對裝置要求較高
- **檔案較大**：相比 Lottie 和 Rive（通常 > 1MB）

### 適合
產品展示（3D 產品旋轉）、Hero 區塊的 3D 背景、互動 3D 場景、品牌視覺頁面。

[Spline - 瀏覽器端的 3D 設計工具](https://spline.design/)

## Spine

### 是什麼？
Spine 是專業的 2D 骨骼動畫工具，主要用於遊戲開發，但在 Web 動畫領域也越來越常見。

### 優點
- **骨骼動畫**：高效的骨骼系統
- **強大的形變**：網格形變讓動作流暢自然
- **檔案小**：骨骼資料緊湊
- **執行效能優秀**：針對遊戲高度優化
- **專業功能**：IK（反向運動學）、約束、換膚、插槽

### 缺點
- **付費軟體**：沒有免費版，需購買授權（$69–$329）
- **學習曲線較陡**：比 Lottie 或 Rive 複雜
- **以遊戲為導向**：主要為遊戲引擎設計（Unity、Unreal、Cocos2d）
- **Web 支援有限**：需要 spine-ts runtime 函式庫

### 適合
2D 遊戲角色動畫、複雜的骨骼角色動作、需要動畫混合的遊戲、高效能 2D 動畫。

[Spine - 2D 骨骼動畫軟體](http://esotericsoftware.com/)

## 效能比較

| 工具 | 檔案大小 | 載入速度 | 渲染效能 |
|------|----------|----------|---------|
| **Lottie** | 最小（< 100KB） | 最快 | 優秀 |
| **Rive** | 小（< 500KB） | 快 | 良好 |
| **Spline** | 大（> 1MB） | 中等 | 取決於裝置 |
| **Spine** | 小（< 200KB） | 快 | 優秀 |

## 如何選擇？

### 選 Lottie 如果你：
- 需要輕量的 2D 動畫
- 需要完整的跨平台相容性
- 已在使用 After Effects
- 需要大量現成動畫資源
- 不需要複雜的使用者互動

### 選 Rive 如果你：
- 需要使用者互動（點擊、拖曳、捲動）
- 需要回應狀態變化的動畫
- 在開發遊戲或 App 微互動
- 可以在 3 個免費專案內完成，或願意付費
- 需要更彈性的動畫控制

### 選 Spline 如果你：
- 需要 3D 動畫效果
- 需要 3D 產品展示
- 不介意全版面顯示
- 目標裝置效能足夠
- 追求高視覺衝擊

### 選 Spine 如果你：
- 在開發 2D 遊戲
- 需要複雜的角色動畫
- 需要骨骼動畫系統
- 需要動畫混合與轉場
- 需要高效能 2D 動畫

## 選擇的核心問題

沒有「最好的」工具，只有適合當下需求的工具。選之前先釐清這幾點：

1. **動畫類型**：2D 還是 3D？靜態播放還是骨骼驅動？
2. **互動性**：需要使用者互動嗎？
3. **效能**：目標裝置的能力如何？
4. **透明背景**：需要與頁面背景融合嗎？
5. **平台**：Web、行動端，還是遊戲？

輕量 2D → **Lottie**；互動 2D → **Rive**；3D 視覺 → **Spline**；2D 遊戲角色 → **Spine**。
