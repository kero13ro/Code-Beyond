這是 [Eric](https://www.linkedin.com/in/eric-liao-754b07b3/) 的技術 blog，分成兩個主軸：

- **🏗️ 如何打造產品：** 團隊流程、系統設計決策，以及工程取捨
- **⚡ 深入開發細節：** 前端效能、瀏覽器底層原理，以及動手實作

## 關於我

我是一名資深前端工程師，從新創到大型企業，涵蓋電商、遊戲、視訊會議與開發者工具。

### 核心技術棧

- **前端：** React、Vue 3、TypeScript、Next.js、Nuxt
- **架構：** Monorepo（Nx / Turborepo）、Micro Frontends、Design Systems
- **效能：** Core Web Vitals 優化、Chrome DevTools profiling、bundle 分析
- **測試：** Playwright、Vitest、Testing Library、視覺回歸測試（Chromatic）
- **DevOps：** CI/CD pipelines、GitHub Actions、Docker、自動化部署
- **AI 整合：** MCP servers、Claude Code 工作流程、prompt engineering

### 我的工程觀

打造好產品不只是寫出乾淨的程式碼——需要同時從三個視角思考：

- **使用者視角：** 怎麼讓產品更直覺、更有價值？
- **團隊視角：** 怎麼協作更順暢、交付更快？
- **商業視角：** 怎麼創造可持續、可量化的價值？

有專案管理與團隊運營的經驗，我關注的是讓工程決策與商業目標對齊。
